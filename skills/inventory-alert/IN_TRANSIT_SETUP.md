# 在途库存配置指南

## 概述

程序支持从飞书多维表格读取在途库存数据，以便更准确地计算建议采购数量。

## 配置步骤

### 1. 创建在途库存飞书表格

在飞书中创建一个多维表格，包含以下字段：

| 字段名 | 类型 | 说明 | 必填 |
|--------|------|------|------|
| SKU编码 | 文本 | 产品 SKU 编码 | ✓ |
| 在途数量 | 数字 | 在途库存数量 | ✓ |

**字段名称支持以下别名：**
- SKU编码：`SKU编码`、`sku`
- 在途数量：`在途数量`、`在途库存`、`quantity`

### 2. 获取表格配置信息

1. 打开飞书多维表格
2. 从 URL 中获取配置信息：
   ```
   https://xxx.feishu.cn/base/[app_token]?table=[table_id]
   ```
   - `app_token`：多维表格的应用 token
   - `table_id`：数据表的 ID

### 3. 配置 alert_config.json

在 `config/alert_config.json` 中添加以下配置：

```json
{
  "feishu": {
    "in_transit_bitable_app_token": "你的在途库存表格 app_token",
    "in_transit_bitable_table_id": "你的在途库存表格 table_id"
  }
}
```

### 4. 验证配置

运行库存预警程序，如果配置正确，会看到类似输出：

```
✓ 已加载 X 个 SKU 的在途库存数据
```

## 计算逻辑

配置在途库存后，建议采购数量的计算公式为：

```
补货数量 = [预计日均销量 × (采购周期 + 物流周期 + 安全预留天数)] - (当前库存 + 在途库存)
```

其中：
- 采购周期：3 天
- 物流周期：12 天
- 安全预留天数：2 天
- 在途库存：从飞书表格读取，如果未配置或读取失败则默认为 0

## 示例

### 在途库存表格示例

| SKU编码 | 在途数量 |
|---------|----------|
| ABC-001 | 100 |
| ABC-002 | 50 |
| ABC-003 | 200 |

### 计算示例

假设某个 SKU：
- 日均销量：10 件/天
- 当前库存：50 件
- 在途库存：100 件（从表格读取）

计算过程：
1. 总周期天数 = 3 + 12 + 2 = 17 天
2. 所需库存 = 10 × 17 = 170 件
3. 建议采购数量 = 170 - (50 + 100) = 20 件

## 注意事项

1. **可选配置**：在途库存配置是可选的，如果不配置，程序会使用默认值 0
2. **表格权限**：确保 OpenClaw 有权限访问在途库存表格
3. **数据更新**：每次运行程序时会重新读取在途库存数据
4. **字段匹配**：程序会尝试多个字段名称，确保至少使用一个支持的字段名
5. **数据验证**：只有在途数量 > 0 的记录才会被使用

## 故障排查

### 问题：未看到"已加载在途库存数据"的提示

**可能原因：**
- 未配置 `in_transit_bitable_app_token` 或 `in_transit_bitable_table_id`
- 配置的值为空字符串

**解决方法：**
检查配置文件，确保两个字段都已正确填写

### 问题：提示"读取在途库存表格失败"

**可能原因：**
- app_token 或 table_id 不正确
- OpenClaw 没有访问权限
- 网络连接问题

**解决方法：**
1. 检查 app_token 和 table_id 是否正确
2. 确认 OpenClaw 有权限访问该表格
3. 检查网络连接

### 问题：加载的 SKU 数量为 0

**可能原因：**
- 表格中没有数据
- 字段名称不匹配
- 在途数量都为 0 或负数

**解决方法：**
1. 检查表格中是否有数据
2. 确认字段名称使用支持的名称（见上文）
3. 确认在途数量 > 0
