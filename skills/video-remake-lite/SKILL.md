# Video Remake Lite

## 核心能力

当用户发来飞书多维表格链接时，运行这条流水线：

1. 读取表格里状态为 `待开始` 的记录
2. 从表格中提取 `视频`、`店铺ID`、`目标国家`、`目标语言`、`商品类型`、可选 `复刻模式`
3. 将视频直接传给模型分析
4. 顺序生成并回写：
   - `高光DNA提取结果`
   - `轻微改写复刻方案`
   - `最终固定分镜`
   - `负面限制词`
5. 任务完成后把状态改为 `已完成`
6. 如果存在 `同步状态` 字段，会写为 `待同步`

当前三步职责：

1. `高光DNA提取结果`
   判断素材是否适合养号复刻，锁定原视频高光、钩子、动作、情绪、BGM 节奏骨架和卡点关系
2. `轻微改写复刻方案`
   只做轻微本地化和防判重改写，不重新创作、不输出多个方向，并保留原视频声音/字幕/动作同步关系
3. `最终固定分镜`
   输出唯一执行分镜和负面限制词；分镜包含情绪目标、音频/节奏要求、卡点动作，可直接进入后续短视频生成任务表

## 时长决策（Step1 强制输出）

下游 AI 视频生成模型当前最长只能生成 15 秒视频。Step1 必须先识别原视频时长并按下表决策；
决策结果会写入 `视频时长` 字段（与下游同步脚本字段名严格一致），**该字段会被同步到原始脚本管理表，由视频生成模型直接消费**。

| 原视频时长 | 复刻策略 | 复刻目标时长 | 任务状态 |
|---|---|---|---|
| < 6 秒 | 不建议复刻 | 0 | `不适合复刻` |
| 6 - 15 秒 | 原长复刻（保留原节奏） | = 原视频时长 | 正常走 Step2/Step3 |
| 15 - 25 秒 | 选段压缩复刻（选 13-16 秒高光段） | 15 秒 | 正常走 Step2/Step3 |
| > 25 秒 | 不建议复刻（信息密度差距过大） | 0 | `不适合复刻` |

实现细节：
- Step1 prompt 顶部新增「零、时长决策」段，模型必须最先输出
- run_pipeline 解析后立即回写 `视频时长(秒)`
- 若策略为「不建议复刻」，跳过 Step2/3，状态置为 `不适合复刻`，原因写入错误信息字段
- Step2 各分段、Step3 表格的镜头时长之和都必须 = 复刻目标时长（偏差 ≤ 7%，绝对偏差 ≤ 1 秒）
- Step3 表格新增「对应原视频时间」列，便于事后人工抽检高保真度

## 声音与情绪保真

高保真轻量复刻默认把 BGM、字幕、动作和人物情绪看作同一套节奏系统：

- Step1 会提取 `BGM / 音频 DNA` 和 `人物情绪与微动作 DNA`
- Step2 会明确哪些 BGM、字幕、动作卡点和情绪曲线必须保留
- Step3 最终分镜必须写出 `情绪目标`、`音频/节奏要求`、`卡点动作`
- 如果原视频是静默字幕型，不会为了讲清楚强行加口播
- 如果原视频是口播型，口播必须像真实人在场景里说话，不能写成解释句或广告句

## 模型约束

本 skill 默认复用 `creator_crm` 视频评分链路的模型配置：

- API URL: `https://ark.cn-beijing.volces.com/api/coding/v3`
- 模型: `Doubao-Seed-2.0-pro`
- 超时: `120s`
- 重试: `2`
- 单步 `max_tokens`: `2500`

也就是说，这个 skill 的视频分析和文本生成都走同一套模型与参数口径。

## 默认字段

必需字段：

- `状态`（单选，**必须包含选项**：`待开始` / `处理中` / `已完成` / `失败` / `不适合复刻`）
- `视频`
- `高光DNA提取结果`
- `轻微改写复刻方案`
- `最终固定分镜`
- `视频时长`（数字，Step1 决策后回写；同步到原始脚本管理表后供视频生成模型消费。命名必须与下游同步脚本一致，**不要带括号或单位后缀**，否则同步链路会取不到该字段）

可选字段：

- `内容分支`，为空时默认 `非商品展示型`
- `店铺ID`，同步到原创脚本表后用于自动发布按账号店铺匹配
- `目标国家`
- `目标语言`
- `商品类型`
- `负面限制词`
- `同步状态`
- `同步到脚本ID`
- `错误信息`

也兼容英文字段名：

- `status`
- `source_video`
- `target_country`
- `target_language`
- `product_type`
- `replicate_mode`
- `analysis_result`
- `replicate_card`
- `localized_script`
- `final_execution_prompt`
- `run_log`
- `video_duration` / `duration`（视频时长字段的英文别名，与同步脚本一致）

别名也支持自动识别，例如：

- `任务状态` / `执行状态`
- `视频素材` / `原视频`
- `产品类型` / `品类`
- `失败原因`

## 执行说明

- 三个生成步骤是单条记录内串行执行
- 每一步 prompt 前都会自动注入“养号高保真轻量复刻”全局总控约束
- 任一记录失败，不会阻塞下一条记录
- 如果表里存在 `错误信息` 或 `失败原因` 字段，会自动写回报错摘要
- 视频字段既支持公网 URL，也支持飞书附件；附件会先换成临时下载链接再传给模型
- Step1 完成后立即根据时长决策回写 `视频时长`；若策略为「不建议复刻」则跳过 Step2/3，状态置为 `不适合复刻`，统计上既不算成功也不算失败
- 若飞书状态字段缺少 `不适合复刻` 选项，会降级为 `失败`，并在错误信息字段前缀 `[不适合复刻]`，便于运维识别后补建选项
