## 🚀 快速开始

### OpenClaw 中使用（推荐）

在 OpenClaw 中说：
```
"处理飞书表格中的待开始任务"
```

OpenClaw 会执行：
```python

subprocess.run(["python3", "process_grid_task.py"])
```

### 命令行使用

```bash
python3 process_grid_task.py
```

## 📋 功能说明

### 1. 读取飞书任务
- 从飞书多维表格读取状态为"待开始"的任务
- 自动跳过"已完成"和"生成失败"的任务

### 2. 处理任务
- 获取前 24 个视频封面（有多少抓多少）
- 获取视频播放量和带货金额数据
- 生成两张宫图（每张 12 个视频）
- 封面上叠加播放量和带货金额

### 3. 更新状态
- 成功 → 更新为"已完成"
- 失败 → 更新为"生成失败"（附带错误信息）

## 🔄 工作流程

```
1. 打开飞书多维表格
   ↓
2. 找到状态为"待开始"的任务
   ↓
3. 逐个处理任务
   ├─ 打开 Kalodata_URL 链接
   ├─ 从 Kalodata 页面获取视频数据
   │  ├─ 视频 ID（前 24 个）
   │  ├─ 播放量
   │  └─ 带货金额
   ├─ 获取视频封面（前 24 个）
   ├─ 生成两张宫图（每张 12 个）
   │  └─ 封面上叠加播放量和带货金额
   └─ 上传到飞书
   ↓
4. 更新任务状态
   ├─ 成功 → "已完成"
   └─ 失败 → "生成失败"（附带错误信息）
```

## 📊 飞书表格字段

### 必需字段
- **TikTok账号**：达人的 TikTok 用户名
- **Kalodata_URL**：Kalodata 达人详情页链接（如：https://www.kalodata.com/creator/detail?id=xxx）
- **状态**：待开始/已完成/生成失败

### 可选字段
- **错误信息**：失败时的错误描述
- **更新时间**：最后更新时间
- **视频截图**：生成的宫图（自动上传）

### 字段说明
- **Kalodata_URL**：OpenClaw 会打开这个链接，从 Kalodata 页面获取视频数据（视频ID、播放量、带货金额）
- **状态**：
  - `待开始`：等待处理的任务
  - `已完成`：成功生成宫图并上传
  - `生成失败`：处理失败（会记录错误信息）

## 🎯 与旧文件的区别

| 功能 | 旧文件 | 新文件 (process_grid_task.py) |
|------|--------|-------------------------------|
| 视频数量 | 只获取 12 个 | 获取前 24 个 |
| 宫图数量 | 1 张 | 2 张（每张 12 个） |
| 视频数据 | 无 | 有（播放量、带货金额） |
| 状态管理 | 无 | 有（待开始/已完成/生成失败） |
| 错误处理 | 简单 | 完整（记录错误信息） |

## 🔍 故障排查

### 问题 1: 没有找到待开始的任务

**原因**：飞书表格中没有状态为"待开始"的记录

**解决**：在飞书表格中添加新记录，并设置状态为"待开始"

### 问题 2: 只获取了 12 张封面

**原因**：使用了旧文件

**解决**：确保使用 [`process_grid_task.py`](process_grid_task.py:1)，不要使用其他文件

### 问题 3: 没有播放量和带货金额

**原因**：
1. 飞书表格中没有提供 `video_data` 字段
2. 或者数据源没有返回这些数据

**解决**：
1. 在飞书表格中添加 `video_data` 字段
2. 或者在代码中实现数据获取逻辑
